4. Tabla BITACORAS
  BITACORAS (
     id          BIGINT UNSIGNED PK AUTO_INCREMENT,
     user_id     BIGINT UNSIGNED     NULL,
     modulo      VARCHAR(50)         NOT NULL,
     entidad_id  BIGINT UNSIGNED     NULL,
     accion      VARCHAR(100)        NOT NULL,
     resultado   VARCHAR(20)         NOT NULL DEFAULT 'exito',
     descripcion TEXT                NULL,
     created_at  TIMESTAMP           NULL,
     updated_at  TIMESTAMP           NULL,

     CONSTRAINT fk_bitacoras_user
        FOREIGN KEY (user_id)
        REFERENCES users(id)
        ON DELETE SET NULL
  );

6. Tabla SESSIONS
  SESSIONS (
     id            VARCHAR(255)    PRIMARY KEY,
     user_id       BIGINT UNSIGNED NULL,
     ip_address    VARCHAR(45)     NULL,
     user_agent    TEXT            NULL,
     payload       LONGTEXT        NOT NULL,
     last_activity INT             NOT NULL,

     INDEX sessions_user_id_index (user_id),
     INDEX sessions_last_activity_index (last_activity)
  );

7. Restricciones funcionales (lógica de aplicación)

  - Roles:
    * admin = control total.
    * user (operador) = acceso según permissions.

  - Permisos granulares (users.permissions):
    * bienes.ver
    * bienes.crear
    * bienes.editar
    * bienes.eliminar
    * categorias.ver
    * categorias.gestionar
    * reportes.exportar

  - Reglas de seguridad sobre administradores:
    * Solo admin puede asignar rol admin.
    * No se permite eliminar o degradar al último admin.

8. Reglas de validación actuales en alta de usuario (backend/frontend)

  - name: min 3 / max 30
  - email: max 60
  - password: min 16 / max 25
  - security_color_answer: min 2 / max 30
  - security_animal_answer: min 2 / max 30
  - security_padre_answer: min 2 / max 30

  Además: sanitización anti HTML (strip_tags), normalización de espacios,
  regex de formato y validación de permisos permitidos.
