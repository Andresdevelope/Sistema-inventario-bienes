# Casos de Uso del Sistema de Inventario de Bienes

**Fecha:** 13 de marzo de 2026

## Introducción

Este documento explica, de manera sencilla y con ejemplos, cómo funciona el sistema de inventario de bienes para cada tipo de usuario. Aquí podrás ver qué puede hacer cada persona según su rol, usando situaciones reales y un lenguaje fácil de entender.

---

## 1. Iniciar sesión en el sistema

**¿Quién lo usa?**
- Cualquier persona autorizada (administrador u operador)

**¿Para qué sirve?**
- Para entrar al sistema usando tu usuario y contraseña.

**Ejemplo práctico:**
- María, que es operadora, abre el sistema, pone su usuario y contraseña, y accede a la plataforma para trabajar.

**Notas importantes:**
- Si te equivocas varias veces con la contraseña, tu cuenta se bloquea y debes pedir ayuda al administrador.

---

## 2. Gestionar bienes

**¿Quién lo usa?**
- Administrador
- Operador de inventario

**¿Para qué sirve?**
- Para registrar, actualizar, buscar o dar de baja los bienes (equipos, muebles, etc.) de la institución.

**Ejemplo práctico:**
- Pedro, operador, registra una nueva computadora en el inventario llenando un formulario sencillo.
- Si necesita actualizar la información de una silla, busca el bien y edita los datos.

**Notas importantes:**
- No se pueden registrar dos bienes con el mismo código.
- Solo quienes tienen permiso pueden hacer estos cambios.

---

## 3. Gestionar catálogos de bienes

**¿Quién lo usa?**
- Administrador
- Operador con permisos especiales

**¿Para qué sirve?**
- Para organizar y actualizar las categorías (por ejemplo, computadoras, escritorios) y ubicaciones (oficinas, almacén) de los bienes.

**Ejemplo práctico:**
- Ana, administradora, agrega una nueva categoría llamada “Proyectores” para que los operadores puedan clasificar mejor los bienes.

**Notas importantes:**
- No se pueden repetir nombres de categorías o ubicaciones.
- Si una ubicación ya no se usa, se puede inactivar para que no aparezca en nuevos registros.

---

## 4. Gestionar usuarios

**¿Quién lo usa?**
- Solo el administrador del sistema

**¿Para qué sirve?**
- Para crear nuevos usuarios, actualizar sus datos o desactivarlos si es necesario.

**Ejemplo práctico:**
- El administrador crea una cuenta para un nuevo operador, o actualiza el correo de un usuario existente.

**Notas importantes:**
- Solo puede haber dos administradores.
- El usuario principal (superadministrador) no puede ser eliminado.

---

## 5. Consultar bitácora de acciones

**¿Quién lo usa?**
- Solo el administrador

**¿Para qué sirve?**
- Para ver un historial de todas las acciones importantes realizadas en el sistema (quién hizo qué y cuándo).

**Ejemplo práctico:**
- El administrador revisa quién eliminó un bien o quién intentó acceder sin permiso.

**Notas importantes:**
- La bitácora es solo para consulta, no se puede modificar ni borrar.

---

## 6. Consultar dashboard por rol

**¿Quién lo usa?**
- Administrador
- Operador

**¿Para qué sirve?**
- Para ver un resumen visual de la información más importante, adaptada según el tipo de usuario.

**Ejemplo práctico:**
- El administrador ve estadísticas completas y actividad reciente.
- El operador solo ve información general, sin datos sensibles.

---

## 7. Recuperar contraseña por preguntas de seguridad


**¿Quién lo usa?**
- Cualquier usuario

**¿Para qué sirve?**
- Para recuperar el acceso si olvidaste tu contraseña, respondiendo preguntas de seguridad y validando un token enviado a tu correo.

**Ejemplo práctico:**
- Si Juan olvida su contraseña, responde correctamente a preguntas como “¿Cuál es tu color favorito?”. Luego, el sistema le envía un token de verificación a su correo electrónico. Juan ingresa ese token en el sistema y, si es correcto, puede crear una nueva contraseña.

**Notas importantes:**
- Las respuestas deben coincidir con las que registraste al crear tu cuenta.
- El token enviado al correo es obligatorio para finalizar la recuperación y tiene un tiempo de validez limitado por seguridad.
